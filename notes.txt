# TinyWebServer 项目笔记 - 增量更新日志

## 当前进度
### 已完成功能
1. **多 Reactor 架构**：
   - Main Reactor (Acceptor) 监听端口，accept 新连接
   - Sub Reactor 线程池管理，每个线程独立 EventLoop
   - 一个 fd 只属于一个 EventLoop 的原则

2. **事件驱动核心**：
   - EventLoop 封装 epoll_wait + 事件回调派发
   - Channel 抽象（fd + 事件 + 回调）
   - 边缘触发（ET）模式支持

3. **Connection 生命周期**：
   - TCP 连接状态机基础框架
   - 非阻塞 I/O，正确处理 EAGAIN/EWOULDBLOCK
   - 读写缓冲区管理（buffer_chain）

4. **HTTP 协议处理**：
   - HTTP 请求解析器（http_parser）
   - 静态资源管理器（static_resource_manager）
   - HTTP 响应生成（http_response）

5. **基础设施**：
   - 异步日志系统（async_logger）
   - 线程池（thread_pool）
   - 定时器管理器（timer_manager）
   - 服务器指标收集（server_metrics）

6. **工程化工具**：
   - CMake 构建系统（支持 Debug/Release、ASan、测试分类）
   - Python 工具脚本（tools.py）用于清理、构建、测试、调试
   - CI/CD 流水线（GitHub Actions，包含 ASan 烟雾测试和 Valgrind 内存检查）

### 已完成功能（新增）
7. **Connection 完整状态机** (v7.1)：
   - 7个状态枚举：Connecting、Connected、Reading、Processing、Writing、Closing、Closed
   - 状态转移验证（CanTransition）与安全转换（Transition）
   - 原子操作保证线程安全，错误转移自动关闭连接

8. **资源边界控制** (v7.1)：
   - ConnectionLimits结构体：64KB输入缓冲区、1MB输出缓冲区限制
   - 实时边界检查（CheckInputBufferLimit、CheckOutputBufferLimit）
   - 背压机制：缓冲区满时暂停读取（PauseReading/ResumeReading）
   - 发送前预检查防止内存溢出

## 关键决策
### 2026-03-06：CI 流水线优化与测试分类
**刚才解决了什么复杂的逻辑？**
- 重构了 `.github/workflows/ci.yml`，将单一任务拆分为两个独立任务：
  1. `build-and-test`：使用 ASan 快速运行冒烟测试（test_timer、test_basic、test_multithread_reactor、test_log）
  2. `memcheck`：禁用 ASan，使用 Valgrind 进行内存泄漏检测
- 在 `CMakeLists.txt` 中引入了 CTest 集成，为测试添加标签（smoke、integration、benchmark）
- 设置了测试超时（30秒）和分类运行机制

**为什么这么改？**
1. **分离关注点**：ASan 和 Valgrind 不能同时使用，拆分后可以充分发挥两者优势
2. **加速 CI**：冒烟测试快速反馈，内存检查作为后续深度验证
3. **可维护性**：测试分类便于选择性运行（开发时跑 smoke，发布前跑全部）
4. **错误隔离**：Valgrind 日志单独保存，便于分析内存问题

## 核心约束（重申）
1. **一个 fd 只属于一个 EventLoop**：禁止跨线程迁移 fd，简化并发模型
2. **边缘触发（ET）**：所有 socket 操作必须使用 ET 模式，循环读写直到 EAGAIN
3. **非阻塞 I/O**：所有 fd 必须设置为 O_NONBLOCK
4. **RAII 资源管理**：每个资源必须有明确的拥有者和生命周期
5. **错误处理**：所有系统调用必须检查返回值，错误信息必须记录
