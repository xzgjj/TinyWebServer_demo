# 测试


## Smoke Test 




test_timer	验证非活动连接的清理或超时逻辑。0.18s 的响应说明时间轮或小根堆算法运行正常。	PASS
test_multithread_reactor	核心并发模型测试。验证了 Main-Reactor / Sub-Reactor 多线程分发是否会导致死锁或竞态。	PASS
test_basic	基础工具类测试（如字符串处理、Buffer 缓冲区、HTTP 解析基础等）。	PASS
test_log	日志系统测试。0.75s 耗时相对较长，通常是因为涉及磁盘 I/O 或异步日志队列的刷盘操作。	PASS


## 压力测试：Smoke Test 
验证的是“能跑通”，建议下一步使用 webbench 或 wrk 进行高并发压力测试，观察 multithread_reactor 在极限情况下的表现。

## 内存检测：
在 build 目录下运行 valgrind --leak-check=full ./test_multithread_reactor，确保多线程下没有内存泄漏或非法访问。

## 全量测试：
如果你还有其他非 Smoke 标签的测试，可以尝试去掉 -L smoke 运行 ctest 看看。
